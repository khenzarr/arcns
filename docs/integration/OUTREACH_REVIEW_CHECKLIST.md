# ArcNS Outreach Review Checklists

---

## After Every Outreach Interaction

Run this immediately after any partner contact ‚Äî email sent, DM received, call completed, package delivered.

- [ ] Log the date and action in the partner's tracker entry
- [ ] Note the channel used (email / DM / call / GitHub)
- [ ] Summarize what was said or sent in 1‚Äì2 sentences
- [ ] Update the partner's status in the tracker
- [ ] Record any open questions or blockers the partner raised
- [ ] Set the next action and next action date
- [ ] If a call happened: send a follow-up message with next steps within 24 hours
- [ ] If a package was sent: set a follow-up reminder for 5‚Äì7 days out
- [ ] If partner went cold: mark `stalled` and set a re-engagement date 3‚Äì4 weeks out

## Weekly Outreach Review

Run once per week. Takes 10‚Äì15 minutes.

### 1. Review active partners

- [ ] Open the tracker ‚Äî scan every partner with status other than `closed`
- [ ] For each: is the next action date today or overdue?
- [ ] For each overdue action: do it now or reschedule with a firm date

### 2. Follow-up check

- [ ] Any partner in `contacted` for more than 7 days without a response? ‚Üí Send one follow-up
- [ ] Any partner in `package-sent` for more than 7 days without a response? ‚Üí Send one follow-up
- [ ] Any partner in `in-review` for more than 2 weeks? ‚Üí Check in lightly
- [ ] Any partner in `call-scheduled` ‚Äî is the call still confirmed?

### 3. Stalled partners

- [ ] Any `stalled` partner whose re-engagement date has passed? ‚Üí Attempt re-engagement or close
- [ ] Is there a new trigger (ecosystem news, new contact, new channel) for any stalled partner?

### 4. New targets

- [ ] Any new potential partners identified this week? ‚Üí Add to tracker as `target`
- [ ] Any warm intros available that haven't been used yet?

### 5. Integration progress

- [ ] Any partner in `integrating` ‚Äî check in if it's been more than 2 weeks
- [ ] Any partner close to launch ‚Äî offer pre-launch review
- [ ] Any integration that went live this week ‚Äî update to `closed`, note outcome

### 6. Tracker hygiene

- [ ] Are all status fields current?
- [ ] Are all next action dates set?
- [ ] Are any entries missing log entries from this week's interactions?

### 7. Weekly summary (optional, 2 sentences)

Write a one-line summary of outreach state for your own records:

> Week of [DATE]: [X] active partners, [X] awaiting response, [X] stalled, [X] integrating. Next priority: [one thing].

## Partner Status Quick Reference

| Status | What it means | Typical next action |
|--------|--------------|---------------------|
| `target` | Identified, not contacted | Send first outreach |
| `contacted` | Outreach sent, no reply | Follow up after 7 days |
| `responded` | Partner replied | Send package or schedule call |
| `package-sent` | Full package delivered | Follow up after 7 days |
| `in-review` | Partner reviewing | Check in after 2 weeks |
| `call-scheduled` | Call booked | Prepare agenda, send pre-call ask |
| `integrating` | Partner is building | Check in every 1‚Äì2 weeks, offer support |
| `stalled` | Gone cold | Re-engage after 3‚Äì4 weeks |
| `closed` | Done (live or declined) | Log outcome, no further action |
